- **Meeting-api `/health` no longer flaps under Redis load — the db-writer keyspace SCAN is off the hot path (#893).** The db-writer's self-healing `meeting:*:segments` scan ran every 10s tick; being `O(keyspace)` it saturated Redis and starved the bounded `/health` PING/XINFO/XPENDING past the 5s probe timeout, so Kubernetes restarted healthy pods under load. The `active_meetings` set is authoritative in steady state, so a tick now sweeps that set alone; the reconciliation scan runs only on startup and every `DB_WRITER_RECONCILE_INTERVAL_S` (default 300s).
