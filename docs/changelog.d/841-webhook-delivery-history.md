- **Webhooks: real deliveries now appear in Delivery History (#841).** The core records every per-user webhook delivery outcome (`delivered | queued | suppressed | blocked | failed`, with status code and timestamp) into a queryable per-user ledger, served by `GET /user/webhook/deliveries`. Previously the dashboard's Delivery History read a store nothing wrote real deliveries to, so genuine deliveries were invisible while the receiver kept getting them. Rows carry the target **host only** — never the webhook URL or signing secret. This is the user-facing completion of #815→#817 (outcome → observable → queryable).
