Vexa is an **AI-first knowledge-worker terminal**: think *Claude Code × Outlook*, with a full backend
of meeting bots and an agentic runtime. This site explains **what we are building and why** — from base
principles, through the primitives, to how every feature is just those primitives composed.

## The one idea

There is exactly **one runtime object: a dispatch** — one run of an agent, in an isolated container,
over a person's files. A chat turn, a scheduled routine, an email worker, and the live in-meeting agent
are the *same* object, differing only in **what triggered it**, **which files it sees**, and **what it's
allowed to do**. Everything else is a **file**, a **tool**, a **trigger**, or **plumbing**.

## It already half-exists

The agentic runtime is **not a leap** — Vexa's **meeting bots are browser containers spawned by the
exact runtime an agent uses**. Same machinery, different workload type. See
[Meetings](/capabilities/meetings) and the [Runtime](/primitives/runtime).
