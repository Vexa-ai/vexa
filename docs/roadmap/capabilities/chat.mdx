Chat is a **`now` [dispatch](/architecture/dispatch)**: the message is the entry point, your
[workspace](/primitives/workspace) is mounted `rw` (chat is **trusted** input), and the reply
[streams](/primitives/stream) live. **Continuity is files in the workspace** — the session id + the
conversation transcript are saved there — so the next turn resumes the conversation even though each
message runs in a fresh, isolated container. No special path: chat is just the unit with a message trigger.

## Implemented

`POST /api/chat` (agent-api) builds the `unit.v1` *now*-dispatch, spawns the container, and relays the
dispatch's Stream (`unit:<id>:out`) back as **SSE** — the same source the gateway exposes over `ws.v1`.
The terminal renders each `tool-call` as an op-row and the `commit` as a badge. Code:
`agent_api.api` (`/api/chat`) + `agent_api.adapters.RedisStreamReader`.
