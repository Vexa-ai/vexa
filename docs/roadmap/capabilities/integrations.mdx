---
title: "Integrations"
description: "Email and calendar as integrations — inbound triggers and outbound tools."
---

Email and calendar are [Integrations](/primitives/integration), not backend objects. **Inbound**: a new
email fires the [Scheduler](/primitives/scheduler) if a triage routine is bound. **Outbound**: the agent
reads/drafts via the integration's MCP tool. Because email is **untrusted input**, triage runs
[propose-only](/architecture/governance) — it suggests `record`/`draft`/`send` cards; a human approves;
`send` stays gated.
