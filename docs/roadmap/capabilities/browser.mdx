---
title: "Browser / web"
description: "Agents drive browser containers — the same workload as the meeting bots."
---

When an agent needs the web, it drives a **browser container** — the *same* `browser` workload the
[meeting bots](/capabilities/meetings) use, spawned by the [Runtime](/primitives/runtime) and driven via a
browser-automation tool (MCP). So "the agent browses the web / operates a web app" reuses the proven
meeting-bot stack; no new primitive.
