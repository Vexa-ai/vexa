---
title: "Knowledge agent"
description: "Processes an input through your workspace — updating it, and producing a grounded artifact."
---

A knowledge agent takes an input — a meeting, an email, a question, a document — and runs it **through
your [workspace](/concepts#workspace)**: it reads the relevant existing knowledge, updates the workspace
with what's new, and produces an artifact that is **grounded** in everything you already know.

This is more than a chatbot answering from a prompt. The output is anchored in accumulated context:

- a **meeting report** grounded on what the workspace already holds about the people and the deal;
- a **briefing** that pulls together everything across past meetings and emails on a topic;
- a **research note** that updates the workspace as it learns, so the next agent starts smarter.

Because input from you is trusted, the [agent](/core/agents) both reads and writes the workspace directly
— git is the undo. The knowledge compounds: every run leaves the workspace a little more complete.
