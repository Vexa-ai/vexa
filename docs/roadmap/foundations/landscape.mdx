Every serious player has converged on the **same shape**: a *context layer* + *agents that act on it* +
*governance*. Atlassian said the thesis out loud — agents are limited less by model IQ than by **how much
company context they can reach**. Vexa shares that goal but sits on a **different axis**, and is the **only
open-source one** in the room.

## Two axes in the market

- **Graph + assistant** — **Glean · Atlassian Rovo · Microsoft Copilot**: a proprietary knowledge graph
  with an assistant and agents bolted on top. Delivered as SaaS.
- **Runtime / control plane** — **Palantir AIP · Microsoft Agent 365**: an *engine that runs governed
  agents* as the product, secure and isolated.

**Vexa is on the runtime axis** — the agents *are* the product — not the search/assistant axis.

## What's distinct about Vexa

1. **Meetings as first-class data.** Everyone else is doc/ticket/graph-first; Copilot's Work IQ treats
   meetings as one input among many. Vexa *captures conversations as the primary substrate* — the richest
   enterprise signal, and the one that's actually lost. (See [Meetings](/capabilities/meetings).)
2. **A real agent [runtime](/primitives/runtime), not an assistant.** Governed, isolated, ephemeral agents
   that *do work continuously* — the Palantir-AIP / Agent-365 lane, not the Glean/Rovo "ask-a-question" lane.
3. **Files, not a proprietary graph.** The knowledge substrate is a [git workspace](/primitives/workspace)
   — transparent, portable, diff-able, with no vendor graph to be locked into.
4. **Open-source + self-hostable + air-gapped** — see below.

## Why the license is the moat

<Note>
Palantir's pitch is *"your data stays yours."* Vexa's is stronger: **your data *and* your code stay
yours.** Vexa is **Apache-2.0** — inspectable, forkable, no per-seat lock-in, runnable entirely inside
your own VPC with [BYO inference and zero egress](/architecture/identity-and-trust).
</Note>

For the target buyers — banks, gov, regulated enterprises — an **auditable, open-source agent runtime they
can run air-gapped in their own environment** is a category of one against these peers. Every competitor
here is a closed SaaS or a closed on-prem black box; the data either leaves the building or sits inside an
unauditable vendor binary. Vexa is the only one where the **runtime itself is open** and the deployment is
the customer's. That is the durable distinction, not a feature checkbox.
