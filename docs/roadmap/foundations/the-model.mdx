## One object

The backend has **exactly one domain object: a dispatch** (one run of an agent). It is not 8 services or
a dozen feature-models — it is one thing, composed from the [seven primitives](/primitives/overview).

A dispatch carries: **who** it acts on behalf of and who triggered it ([Identity](/primitives/identity)),
**which agent** runs ([Agent](/primitives/agent)), **which files** it mounts and at what access
([Workspace](/primitives/workspace)), **which tools** it may call ([Integration](/primitives/integration)),
and **what to do** (an entry point, or a session to resume). The [Scheduler](/primitives/scheduler) fires
it; the [Runtime](/primitives/runtime) runs it isolated; it streams on a [Stream](/primitives/stream).

## Everything else is one of four things

<CardGroup cols={2}>
  <Card title="A file">
    Knowledge, plans, tasks, notes, the agent's own session — all just files in a workspace. **No
    dictated structure**, in code or contract.
  </Card>
  <Card title="A tool">
    A capability reached through an [Integration](/primitives/integration) (email, calendar, browser,
    web). Cred-gated. Thousands, all uniform.
  </Card>
  <Card title="A trigger">
    An input to the [Scheduler](/primitives/scheduler): a schedule entry, an integration event, or a
    live message.
  </Card>
  <Card title="Plumbing">
    Runtime, dispatch routing, streaming, identity — the substrate the object runs on.
  </Card>
</CardGroup>

There is **no `email`, `task`, `meeting-note`, or `calendar` object** in the backend. The terminal renders
*views*; the backend runs *dispatches*.

## Every feature is just primitives

| Feature | Is just… |
|---|---|
| **Chat** | a *now* dispatch; the message is the entry point; streamed back |
| **Routines / automations** | a [Scheduler](/primitives/scheduler) entry (time or integration event) |
| **Tasks / notes / entities** | **files** in a [Workspace](/primitives/workspace) |
| **Email / Calendar** | [Integrations](/primitives/integration) — never backend primitives |
| **Meetings** | browser-container bots on the [Runtime](/primitives/runtime) → `transcript.v1` |
| **Live in-meeting** | a dispatch the transcript stream keeps alive, emitting cards on its [Stream](/primitives/stream) |
| **Org knowledge** | a scheduled dispatch merging folders in git |

<Note>
This is the discipline that catches mistakes early: if a proposed feature wants its own backend object,
it is almost certainly wrong — it should be a file, a tool, a trigger, or a dispatch.
</Note>
