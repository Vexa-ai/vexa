# docs/docs/governance — the published law

**Concern:** the governance set, published on docs.vexa.ai (single source of truth): the two
constitutions (`architecture.mdx` P-book · `delivery.mdx` D-book), the two operational protocols
(`delivery-prepare.mdx` · `delivery-take.mdx`), the compliance maps (`delivery-compliance.mdx` ·
`arch-compliance.mdx` — the latter GENERATED by `scripts/arch-report.mjs`, never hand-edited),
and the overview article (`index.mdx`, "How Vexa governs itself").

**Surface:** rendered by Mintlify (nav group "Governance" in `../docs.json`); agent-readable as
`.md` + indexed in the site's `/llms.txt`. Old repo paths (`docs/ARCHITECTURE.md`, …) carry
pointer stubs.

**Deps:** none at runtime; `arch-compliance.mdx` is regenerated by `node scripts/arch-report.mjs`
and checked by `gate:arch-report`.
